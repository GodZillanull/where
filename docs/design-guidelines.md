# デザインガイドライン

## 基本思想：深津流デザイン

本アプリのUI/UXは深津貴之の思想に基づいて設計する。

### 核心原則

> **「ミニマルは手段であって目的ではない」**
> **「ユーザーが"決める"ために必要な情報だけを、最高の形で見せる」**

---

## 5つの設計原則

### 1. 1画面1メッセージ

各画面には「この画面で何を決めるか」が1つだけ存在する。

| 画面 | 決めること | 必要な情報 |
|------|-----------|-----------|
| ホーム | 週末プラン or 寄り道 | 2つの選択肢のみ |
| プラン選択 | どのプランに行くか | 3つの候補 |
| プラン詳細 | このプランで行くか | 体験内容、所要時間、予算 |
| スポット詳細 | ここに寄るか | reason、アクセス、予算 |

**アンチパターン：**
- 1画面に複数のCTAを並べる
- 「ついでに」情報を追加する
- 将来使うかもしれない機能を入れる

### 2. 勝ち筋を最初に見せる

ユーザーが最も知りたいことを画面の最上部に配置する。

```
✅ 良い例
┌─────────────────┐
│ 「静かに本と    │  ← 体験（勝ち筋）
│  過ごせる場所」 │
│                 │
│ 書店 SHIBUYA    │  ← 店名（補足）
└─────────────────┘

❌ 悪い例
┌─────────────────┐
│ 書店 SHIBUYA    │  ← 店名（興味ない）
│ 渋谷区...       │  ← 住所（まだ早い）
│ 営業時間...     │  ← 詳細（決めてから）
│ 静かに本と...   │  ← 体験（埋もれてる）
└─────────────────┘
```

### 3. 段階的開示

情報は「今必要なもの」だけ見せる。詳細は次のステップで。

```
Step 1: 行くか決める → reason、駅、予算
Step 2: 行くと決めた → 営業時間、地図、詳細
Step 3: 現地にいる   → ナビ、電話番号
```

### 4. ノイズを減らす

「あったら便利かも」は入れない。「ないと決められない」だけ入れる。

| 情報 | 必要性 | 理由 |
|------|--------|------|
| reason | ◎ 必須 | 行く理由そのもの |
| 駅・徒歩時間 | ◎ 必須 | 行けるか判断 |
| 予算 | ◎ 必須 | 払えるか判断 |
| 営業時間 | ○ 次画面 | 決めた後に確認 |
| 定休日 | ○ 次画面 | 決めた後に確認 |
| 電話番号 | △ 不要 | 現地で困った時だけ |
| 住所 | △ 不要 | 地図を見ればわかる |

### 5. 感情を動かす

ミニマルでも「ワクワク」は必要。色と言葉で感情を動かす。

## カラーシステム

| タイプ | メインカラー | 背景カラー | テキストカラー |

| カフェ | `#8B7355` | `#F5F0EB` | `#5C4A3A` |

| 銭湯 | `#4A90A4` | `#EBF4F7` | `#2D5A6A` |

### reason の扱い

reasonは「勝ち筋」なので、特別な扱いをする。

```css
/* reasonスタイル */
font-size: 15-17px;
line-height: 1.6;
color: #555555;
/* または引用風に */
font-style: italic;
border-left: 2px solid [type-color];
padding-left: 12px;
```

## 写真の扱い

### 原則

> **「ないものを無理に埋めるな。ある武器で戦え」**

### 判断基準

| 状況 | 対応 |
|------|------|
| 写真データがある | 表示する |
| 写真データがない | 色+タイポグラフィで代替 |
| 汎用プレースホルダー | **使わない**（ノイズになる） |

### 将来的な写真対応

1. **自前撮影** - 品質最高、工数大
2. **Unsplash API** - 雰囲気写真、無料、商用可
3. **Instagram埋め込み** - 非推奨（API制限、認証必要、不安定）

### 「写真を見る」リンク

写真がない場合、Instagramへのリンクで代替する。

```javascript
// Instagram ハッシュタグ検索
`https://www.instagram.com/explore/tags/${encodeURIComponent(spotName)}/`
```

## 絵文字の使用

### 原則

絵文字は「タイプ識別」にのみ使用。装飾には使わない。

| 用途 | 可否 | 例 |
|------|------|-----|
| タイプアイコン | ✅ | ☕ カフェ、📚 書店、♨️ 銭湯 |
| ボタンラベル | ❌ | ~~「行く！🚀」~~ → 「行く」 |
| 見出し装飾 | ❌ | ~~「✨おすすめ✨」~~ → 「おすすめ」 |
| リスト記号 | ❌ | ~~「📍 渋谷駅」~~ → 「渋谷駅」 |

## レイアウト原則

### 余白

```
画面端パディング: 24px (px-6)
セクション間: 24-32px
要素間: 8-16px
```

### 最大幅

```
コンテンツ最大幅: max-w-lg (512px)
中央揃え: mx-auto
```

### CTA固定配置

決断を促す画面では、CTAを画面下部に固定する。

```jsx
<div className="min-h-screen flex flex-col">
  <div className="flex-1">
    {/* スクロール可能なコンテンツ */}
  </div>
  <div className="px-6 pb-8 pt-4">
    {/* 固定CTA */}
  </div>
</div>
```

## 禁止事項

1. **汎用プレースホルダー画像** - 「この店」の魅力を伝えない
2. **装飾目的の絵文字** - ノイズになる
3. **「あったら便利」機能** - 必要になってから追加
4. **複数CTA** - 1画面1アクション
5. **営業時間の初期表示** - 決めた後の情報
6. **星評価・スコア** - 数字より言葉で伝える
