1) 検証内容（2週間のMVP検証で「何を確かめるか」）

具体的な検証項目（混ぜずに見る）
A. 平日（Pull）検証：ユーザー起因で開いた時に“即決→実行”できるか

6) Go/No-Go判断基準（数値で決める）

サンプル30は小さいので、**「強いシグナルが出たらGo」**の基準にします。
（※数字は“意思決定用のガードレール”。達成できないなら改善かピボットが必要）

Gate 0（品質）— ここ落ちたら即No-Go（先に直す）

地図起因の失敗率（cant_find_destination / open_maps）が 2%未満（理想は0%）

主要プランで「目的地が見つからない」が頻発しない

落ちた場合の対応

“抽象名をナビに使っている”のが原因なので、アンカーを全面修正して再試験

ここを直す前に他を議論しても無意味

Gate 1（刺さり：決定→実行）— ここが通ればプロダクトは成立

平日Pullと週末Pushで分けて判定（混ぜない）

平日Pull：意思決定が速いか

start_plan / plan_view が 35%以上

（開いたのに決めない＝刺さってない or 情報過多）

done / start_plan が 25%以上

（決めたのに動けない＝プランが重い or PlanBが弱い）

週末Push：きっかけとして機能するか

plan_view / push_click（=リンクからの到達） 60%以上

done / start_plan 25%以上（実行率は平日と同水準でOK）

どちらか一方だけ良いでもGoはあり得るが、あなたは週末特化にしたくないので
「平日Pullが最低ラインを超えているか」を重く見ます。

Gate 2（継続の芽）— 事業Goの強い根拠

2週間で done が 2回以上の人が 30人中6人以上（20%）

そのうち 平日Pull起因が2回目に含まれる人が2人以上

ここが出ると「改善して伸びる」確度が上がります。

Gate 3（事業性の芽：GTM/収益）— 伸ばし方が見えるか

この段階は“兆し”で十分。

集客の再現性（最低条件）

テスター獲得が「1回限り」じゃなく、同じ手段で繰り返せる

例：広告・イベント・紹介のどれかで
週に10人以上を安定して集める見込みが立つ

収益の兆し（どれか1つでOK）

WTP（支払い意思）テストで

「月額◯円なら払う」or「有料でも使う」が **20%**以上
（実課金でなく“強い同意”でも良いが、言葉だけにしない工夫が必要）

送客価値の兆し

「この店に行った」「この周辺で消費した」ログが取れて、提携の仮説が立つ
